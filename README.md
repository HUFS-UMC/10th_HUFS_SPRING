# 🌱 UMC 10기 HUFS Spring Boot Study

UMC 10기 한국외대 Spring Boot 스터디 공용 레포지토리입니다.  
이 레포는 **각자 본인 폴더 안에서 미션과 프로젝트를 이어가는 방식**으로 운영합니다.

## 📌 작업 흐름

모든 작업은 `main` 브랜치에서 바로 진행하지 않고, **개인 작업 브랜치**를 만들어 진행합니다.

1. 최신 `main` 브랜치를 기준으로 작업 시작

```bash
git checkout main
git pull origin main
```

2. 본인 작업 브랜치 생성

```bash
git checkout -b week1-bodol
```

3. 작업 후 변경사항 커밋

```bash
git add .
git commit -m "feat: 1주차 미션 기본 구조 추가"
```

4. 원격 브랜치로 푸시

```bash
git push origin week1-bodol
```

5. GitHub에서 `main` 브랜치로 PR 생성

- PR은 반드시 **본인 브랜치 -> `main`** 기준으로 생성합니다.
- PR 작성 시 레포에 등록된 템플릿을 사용합니다.

## 🌿 브랜치 네이밍 컨벤션

브랜치명은 아래 형식으로 통일합니다. (주차별로 브랜치가 생성될 예정이므로 통상적으로 사용되는 컨벤션이 아닌 특수한 브랜치 네이밍 컨벤션을 적용하겠습니다!)

```bash
week()-닉네임
```

예시:

```bash
week1-bodol
week2-bodol
```


## 📝 커밋 메시지 컨벤션

커밋 메시지는 아래 형식으로 작성합니다.

```bash
타입: 작업 내용
```

예시:

```bash
feat: 회원가입 API 구현
fix: 로그인 예외 처리 수정
docs: README 작업 흐름 문서화
refactor: 상품 서비스 로직 분리
chore: application.yml 예시 파일 정리
```

## 📁 폴더 작업 규칙

- 각자는 **본인 이름이 포함된 폴더 안에서만** 프로젝트를 생성하고 작업합니다.
- 다른 사람의 폴더나 프로젝트는 수정하지 않습니다.
- 공용 문서나 공통 설정 수정이 필요한 경우에는 변경 이유를 PR에 반드시 남겨주세요.

현재 생성된 개인 폴더:

- `야르_김민준 야르입니다.`
- `에이셉_김현수 파트부원입니다요`
- `버찌_박서연`
- `원호_박원호 (일호엿으면 조켓는디)`
- `레아_이가연 레아_이가연임니다..`
- `리미_이예림`
- `보돌-주보경 파트장입니다요.`
